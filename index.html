Quick start

Junebug needs ripgrep for its search tool (the macOS installer takes care of it) and, to build from source, a Rust toolchain. Prebuilt binaries for macOS (arm64/x86_64), Linux, and Windows ship on every tagged release.

 terminal
$ git clone https://github.com/weeksdev/junebug-cli && cd junebug-cli
$ ./install-macos.sh          # builds, installs ripgrep if needed, puts junebug in ~/.local/bin
$ junebug set --provider deepseek YOUR_API_KEY
saved — starting the REPL…

Keys are stored once in ~/.junebug/credentials.env (mode 0600) — or use environment variables or a workspace .env. With no key at all, the REPL still opens and /keys sets one up in place, with the input hidden. Keys never enter the conversation, the session log, or the model.

Any model, including fully local

Junebug speaks OpenAI-compatible Chat Completions and the Anthropic Messages API natively, with real SSE streaming, tool calls, usage accounting, and mid-session provider switching. One /model menu lists every provider you have credentials for — including a running Ollama — and switches in a single step.

openrouter

Default cloud provider. One key, hundreds of models. OPENROUTER_API_KEY

openai · deepseek

Direct Chat Completions with streamed tool calls; DeepSeek thinking-mode output is handled across provider switches.

anthropic

Native Messages API: SSE text/tool streaming, model discovery, canonical tool_use transcripts.

ollama · local-openai

Fully local, no key. Auto-detects the runtime; LM Studio, vLLM, and llama.cpp work via LOCAL_OPENAI_BASE_URL.

There is also opt-in per-call routing (--model auto): cheap turns go to cheap models, hard turns to strong ones, with visible route notices and safe switching between turns — never mid-tool-call.

Safety is the architecture, not a feature

Every tool call — built-in or MCP — passes through a deterministic PolicyEngine before any side effect runs. Its decision comes from the permission mode and plan mode only: no model output, no tool arguments, no heuristics. The default is read-only.

Cycle modes mid-session with Shift+Tab — even while a turn is running — and a colored badge under the prompt always shows the effective level. --plan is a hard read-only guard on top of everything.

Defense in depth

protected paths

.env*, .git, and .junebug are untouchable at any depth outside yolo, matched per path component — Windows separators and nested entries included.

symlink containment

Symlinks are fully resolved; a link inside the workspace cannot read or create anything outside it. Traversal (..) is rejected.

command hygiene

Destructive/network patterns get a ⚠ warning in the approval prompt (token-based matching, so git add . is fine and ;rm -rf / is not). Commands run with a sanitized environment.

command timeouts

Every command is killed — with its whole process tree — after a model-settable timeout (default 120s). A foreground server can't hang a turn.

A real terminal UI

Streamed Markdown with ANSI styling, a spinner with elapsed time, ⏺ tool(args) / ⎿ result activity lines, and Esc to interrupt a running turn cleanly — partial tool calls are discarded and the transcript stays valid.

completions

/ opens the command menu; @ searches workspace files and attaches them to your message.

/explorer

Searchable workspace tree with a syntax-highlighted viewer. Press e to open the file in your own $EDITOR — checkpointed first, so even manual edits are rewindable.

/changes · /diff

Full-screen changed-file browser with per-file diffs — real Git when present, the latest shadow checkpoint otherwise.

file diffs

Every write is shown as a colored diff hunk, embedded in approval prompts in ask mode. Diffs are UI-only and never sent to the model.

/compact

Model-written summary replaces the in-memory history to free context; the session log keeps the full record.

/model · /keys · /permissions

Arrow-key menus for switching models across all configured providers, saving keys with hidden input, and changing permissions live.

Checkpoints: undo for agents

Before every prompt and before every mutating tool — including shell commands — Junebug snapshots the workspace into a shadow Git repository under ~/.junebug/checkpoints/. Your own repo is never touched, and non-Git workspaces work fine.

/rewind lists checkpoints in a menu and restores any of them; the pre-restore state is checkpointed first, so a rewind is itself undoable. Snapshots respect your .gitignore and never capture .env*, .junebug/, or build caches — proven by tests that secrets are neither captured nor overwritten on restore.

Why it matters: file-write checkpoints are easy. Checkpointing before commands is what saves you when a script the agent ran rewrites half the project.

Model swarms

One strong model plans and judges; cheap models do the work; nothing is accepted unverified. That's /swarm.

boss plans (read-only)→ worker executes→ checker verifies→ boss reviews

The boss inspects the workspace under a hard read-only guard and writes a constitution plus a task list. For each task, a worker gets full tools and a checker independently re-verifies with its own tools — it cannot write files and must end with an explicit VERDICT: line, where silence counts as failure. Failed checks send specific feedback back to the worker; persistent disputes escalate to the boss, whose unclear rulings default to the checker, so unverified work is never accepted.

 junebug — /swarm
❯ /swarm Add todo CRUD endpoints and a minimal HTML page

  ✗ check failed: page does not refresh the list after adding

Built for long runs

Transient provider errors (a stream dying mid-turn, a 502, a rate limit) retry the affected agent turn with backoff instead of killing the run. Progress is saved after planning and after every task, so if a swarm does abort, /swarm resume continues exactly where it left off. Checkpoints are taken throughout — /rewind can undo an entire swarm. Mix providers freely: a cloud boss with local Ollama workers is a fine (and cheap) setup, assigned in one /swarm-setup menu.

Sessions, resume, and headless mode

Every conversation is recorded as JSON Lines under .junebug/sessions/ — structured messages plus an audit trail of tool calls, route switches, checkpoints, and swarm events. --resume with no path lists this workspace's conversations newest-first with a preview; --resume-compact summarizes a large history before the first turn so resuming a 28 KB session costs ~1.4k input tokens instead of ~7k.

For scripts and CI there's junebug exec --json "prompt": newline-delimited events (text.delta, tool.call, tool.result, file.diff, completed) on stdout, serialized with stable keys.

Local stdio MCP servers (--enable-mcp) and lifecycle hooks (--enable-hooks) are off unless you turn them on, and MCP tool calls pass through the same policy engine as built-ins.

FAQ

Is my code sent anywhere?

Only to the provider you choose, over HTTPS — and with Ollama or a local OpenAI-compatible server, nothing leaves your machine at all. Sessions, checkpoints, credentials, and configuration are all local files.

What keeps the agent from wrecking my machine?

The default mode can't write or run anything. Above that: a deterministic policy gate on every tool call, protected paths, symlink containment, command warnings and timeouts, approval prompts with diffs, and checkpoints before every mutation so /rewind can undo it. Yolo mode exists, but you have to ask for it and it tells you exactly what you're giving up.

What do I need installed?

ripgrep (the installer handles it on macOS; Junebug warns at startup if it's missing). Git is optional. Rust only if you build from source.

Where did the name come from?

Junebug is the tortoiseshell cat in the logo. The project was previously called Febo; legacy .febo state stays readable.
